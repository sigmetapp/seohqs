# Проверка верификации домена в Resend

Если Resend API настроен, но письма не приходят, скорее всего проблема в верификации домена.

## Проблема

Когда вы используете `noreply@seohqs.com` или другой email с вашим доменом, Resend требует, чтобы домен был верифицирован.

## Как проверить статус домена

1. Откройте https://resend.com/dashboard
2. Перейдите в раздел **"Domains"**
3. Найдите домен `seohqs.com`
4. Проверьте статус:
   - ✅ **Verified** - домен верифицирован, все должно работать
   - ⚠️ **Pending** - домен в процессе верификации
   - ❌ **Unverified** - домен не верифицирован

## Если домен не верифицирован

### Вариант 1: Верифицировать домен (рекомендуется)

1. В Resend Dashboard перейдите в **"Domains"**
2. Добавьте домен `seohqs.com` (если еще не добавлен)
3. Добавьте DNS записи в настройках вашего домена:
   - **SPF**: `v=spf1 include:resend.com ~all`
   - **DKIM**: (будет предоставлен Resend)
   - **DMARC**: `v=DMARC1; p=none;` (опционально)
4. Дождитесь верификации (обычно несколько минут)

### Вариант 2: Временно использовать onboarding@resend.dev

Если нужно быстро решить проблему, временно измените переменную окружения:

```env
RESEND_FROM_EMAIL=onboarding@resend.dev
```

**Важно:** После этого перезапустите деплоймент в Vercel.

Код автоматически попробует использовать `onboarding@resend.dev` если отправка с вашего домена не удалась, но лучше настроить правильно.

## Проверка через API

После настройки проверьте работу:

```bash
# Тестовая отправка
curl -X POST https://seohqs.vercel.app/api/admin/email-diagnostics \
  -H "Content-Type: application/json" \
  -d '{"email": "ваш@email.com"}'
```

## Проверка логов в Resend

1. Откройте https://resend.com/dashboard
2. Перейдите в **"Logs"** или **"Emails"**
3. Найдите записи о попытках отправки
4. Проверьте статус:
   - ✅ **Delivered** - письмо доставлено
   - ⚠️ **Bounced** - письмо не доставлено (проверьте причину)
   - ❌ **Failed** - ошибка отправки (проверьте детали)

## Частые ошибки

### "Domain not verified"
**Решение:** Верифицируйте домен в Resend Dashboard или используйте `onboarding@resend.dev`

### "Invalid from address"
**Решение:** Убедитесь, что email адрес соответствует верифицированному домену

### "Rate limit exceeded"
**Решение:** Проверьте лимиты в Resend Dashboard. Бесплатный план имеет ограничения.

## Рекомендации

1. **Для production:** Всегда верифицируйте домен для лучшей доставляемости
2. **Для тестирования:** Можно использовать `onboarding@resend.dev`
3. **Мониторинг:** Регулярно проверяйте логи в Resend Dashboard
